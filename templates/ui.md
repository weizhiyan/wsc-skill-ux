# UI 界面 / 设计稿生成模板

适用：Stitch、Figma Make、Figma AI、Cursor、Trae、Copilot 等 AI 生成 UI 工具。
参考记忆：默认优先参考 `memory/ui_design_spec_ultra_brief.md`；需要更多细节时再参考 `memory/ui_design_spec_brief.md`；如需严格约束或完整项目设计规范，再参考 `memory/ui_design_spec.md`。

## 适用目标
用于生成：
- 普通 UI 页面
- 后台管理系统页面
- 工作台 / 仪表盘
- 列表页 / 详情页 / 配置页
- SaaS 产品界面
- 完整页面设计稿描述

---

## 输出结构
1. 页面目标
2. 目标用户 / 使用场景
3. 页面整体布局
4. 核心模块拆解
5. 信息层级
6. 视觉风格要求
7. 基础交互要求
8. 可直接给 AI 工具使用的 UI / 设计稿描述

---

## 追问重点
当信息不足时，优先追问：
- 这是后台、工作台、列表页、详情页、官网、活动页，还是某个具体业务页面？
- 页面最重要的 2~3 个模块是什么？
- 更偏简洁、科技、高级、深色、轻量，还是品牌化？
- 你是只要静态设计稿，还是还需要基础交互说明？

---

## 输出要求
- 默认输出适合生成完整 UI 页面或设计稿的描述
- 不要只写视觉风格，必须写清页面结构和模块层级
- 优先体现内容驱动、克制、有留白、信息层级明确的视觉气质
- 中文字体优先使用 **PingFang SC**
- 优先使用自动布局（Auto Layout）组织页面和组件
- 避免无意义、多余的 Frame 包裹
- 页面内容要尽量填充合理区域，避免空、散、松
- 如果是后台 / SaaS 页面，要体现主次关系，不要把所有模块做得一样重
- 若用户提到交互，只补最关键的基础交互；如果明确要动效或前端实现，再转交互动效规则
- 最终给 AI 工具直接使用的描述必须放在代码块中，方便复制

---

## 最终描述建议内容
最终给 AI 工具的描述中，尽量包含：
- 页面类型
- 页面核心目标
- 布局方式
- 模块组成
- 视觉风格
- 配色倾向
- 字体与层级感
- 卡片 / 表格 / 图表 / 按钮等组件要求
- 留白与节奏要求
- 如有需要，再补基础交互说明
