# 模式B：交互优化

> 路径标记格式：`[B > 1/4]`

## 目标
快速定位交互问题类型，给出具体优化建议。

## 追问流程

### [B > 1/4]
> 能描述一下这个操作的完整步骤吗？从你触发它开始，到你想完成的目标结束。
> 为什么问：还原完整操作路径，才能找到问题真正卡在哪一步。

STOP — 等待用户回答

### [B > 2/4]
> 哪一步让你感觉最卡或最难受？
> 为什么问：定位具体断点，避免泛泛而谈。

STOP — 等待用户回答

收到回答后，对照以下表格判断问题类型：

| 问题类型 | 判断标准 |
|---|---|
| 可见性问题 | 找不到按钮/信息被遮挡/不知道当前状态 |
| 控制性问题 | 无法自定义/操作被限制/不能撤销 |
| 反馈问题 | 操作后没反应/不知道有没有成功 |
| 效率问题 | 步骤太多/需要频繁切换/重复操作 |
| 一致性问题 | 同类操作逻辑不统一/与常见软件习惯冲突 |
| 容错问题 | 容易误操作/出错后无法恢复 |

### [B > 3/4]
> 这个问题只在这一个地方出现，还是整个产品里都有类似情况？
> 为什么问：局部问题和全局问题的修复策略完全不同。

STOP — 等待用户回答

### [B > 4/4]
> 用户每天大概要做这个操作几次？
> 为什么问：使用频率决定优化的优先级。

STOP — 等待用户回答，然后输出报告

---

## 输出模板

```
【交互问题定位】
问题类型：[填入对应类型]
问题描述：[一句话描述]
影响范围：[局部/全局]
使用频率：[高频/低频]
影响程度：[低/中/高]

【优化建议】
P0 立即可做：[具体可执行的建议]
P1 进一步优化：[更深入的改进方向]
需要注意：[实施时要避免的坑]
```

## 防呆规则
- 用户描述不清楚时："能具体说说是哪一步让你感觉难受吗？比如点了某个按钮之后发生了什么？"
- 无法判断问题类型时，选最接近的，标注"（推测）"。
