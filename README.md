> 将模糊的产品想法，转化为像素级精准的 UI 设计描述与底层体验逻辑。

直接发你的想法——哪怕是零碎的交互描述、粗糙的页面骨架，或者一句「帮我做个弹窗」——`wsc-ux` 会自动判断场景，整理成结构化规格，或直接生成 Figma Make/Stitch 可用的完整提示词。

它不仅仅是一个排版工具，更是一个**内置了顶级产品方法论（Kano、Nielsen 10、Fitts's Law 等）的资深产品专家**，随时为你做深度需求诊断。

## 核心工作流与三大场景

`wsc-ux` 被严格划分为三个核心场景，无需你手动切换，它会根据你的输入自动路由：

### 场景 1：碎片梳理 (Fragment Structuring)
**当你的脑子里只有一团乱麻，它帮你把“口水话”变成专业文档。**

你只需丢过来一段零散的描述（比如：“这个按钮点一下转圈，成功变绿弹出提示，失败就抖一下”），它会自动按类型输出规范文档：

- **交互状态机**：输出完整的 `Default / Hover / Active / Loading / Disabled` 状态矩阵，包含触发条件与视觉表现。
- **动效规格**：精确定义入场/退出的时序节奏，甚至给出动效的贝塞尔曲线（Cubic-bezier）和推荐时长（如 200ms）。
- **页面布局**：拆解界面的信息层级，明确哪些是主要视觉焦点，哪些是次要辅助信息。
- **功能主流程**：梳理核心目标、主链路步骤、关键决策点，并强制要求补充“边界异常情况”（如网络超时、空数据态、防抖限制等）。

---

### 场景 2：提示词补全 (Prompt Completion)
**做 AI 绘图引擎（Figma Make/Stitch）的“中控大脑”。**

当你给出一个粗糙的页面骨架，说「帮我补全」「做个提示词」时，它会生成一段高质量的长提示词。

**核心工作策略：**
- **功能逻辑“锁死”**：把模块关系、数据联动、真实数据示例、异常状态写得极其严密，绝不允许 AI 在逻辑上犯错。
- **视觉细节“留白”**：故意不写死具体的色值或像素级圆角（除非你明确要求），给下游的 AI 引擎留出发挥空间。
- **平台智能判断**：全页面/弹窗自动指向 `Stitch`；单个组件/卡片自动指向 `Make`。

**独家视觉审美预设（内置基因）：**
*摒弃通用的 AI 美学，注入顶级视觉品味：*
- **极简 iOS 风**：大面积留白，克制的信息密度，基于 4px/8px 严格栅格。
- **高对比色彩**：主打高对比蓝绿配色（主色 `#165DFF`，辅助色 `#00B42A`），视觉焦点极其明确。
- **内敛动效**：偏好克制的“微光呼吸”动效与轻盈反馈，拒绝花哨的弹跳与过度动画。

---

### 场景 3：产品顾问 (Product Consultant) 💡 硬核功能
**当你需要深度的产品诊断与方案推演时激活。**

当你明确说「帮我分析」「评审一下」「竞品分析」时，它不会敷衍地直接给结论，而是**弹窗引导你选择诊断模式**，并加载外部专业模板文件，扮演专家，按步骤向你提问、带你推演。

包含 6 大硬核专家模式：

| 模式 | 解决什么问题 | 内置专家框架 / 方法论 |
| :--- | :--- | :--- |
| **A. 深度分析** | 用户反馈一团乱麻，找不到真实痛点 | 运用 **5 Whys 分析法** 寻找根因，梳理触点漏斗，将问题结构化为“认知/视觉/链路阻断”。 |
| **B. 交互优化** | 功能有了，但用起来“很涩”、转化率低 | 运用 **Fitts's Law (费茨定律)** 与 **Hick's Law (席克定律)** 缩短操作路径，引入内敛微交互减少心智摩擦。 |
| **C. 功能优化** | 老功能二期迭代，或者想“做得更好” | 运用 **Kano 模型** 识别期望属性与魅力属性，进行极致的边界与空状态（Empty State）体检。 |
| **D. 新功能设计** | 只有一个想法，不知道怎么落地成页面 | 构建核心用户故事 (User Story)，规划主链路 (Happy Path)，输出 MVP 最小可行性功能的页面结构。 |
| **E. 方案评审** | 画完草图心里没底，需要总监来“挑刺” | 对照 **尼尔森十大可用性原则 (Nielsen's 10 Heuristics)** 无情纠错，给出逻辑死胡同和体验雷区的“体检报告”。 |
| **F. 竞品分析** | 想知道行业天花板是怎么做的 | 选取顶级标杆（如 Apple, Stripe, Linear）进行逆向工程，提炼差异化最优解，而非盲目抄袭。 |

## 核心机制

### 1. 弹窗引导系统（AskUserQuestion）
场景不明确、或需要用户做决策（特别是场景 3）时，统一通过原生弹窗收集信息，不乱猜，也不在对话框里反复追问纯文字。

### 2. 记忆轴系统（Memory）
每次使用后自动将偏好和规律写入本地记忆轴（`E:\claud文件\对话记录\`），实现真正的“越用越懂你”：
- **设计偏好记忆**：视觉风格、极简偏好、色彩倾向。
- **交互模式记忆**：常用状态机逻辑、动效节奏偏好。
- **历史归档**：历史高质量提示词备份。

## 安装与触发

### 安装
将 `wsc-ux` 文件夹放入 Claude Code 的技能目录：
```bash
~/.claude/skills/wsc-ux/
```

### 触发关键词
```text
# 触发场景 1：碎片梳理
帮我梳理一下 / 补充一下 / 整理一下
这是一个弹窗/页面/交互 / 交互状态 / 动画

# 触发场景 2：提示词补全
做个提示词 / 帮我补全UI / Make / Stitch / 设计稿

# 触发场景 3：产品顾问（重量级）
帮我分析 / 评审一下 / 竞品分析 / 新功能怎么做 / 诊断一下
```
