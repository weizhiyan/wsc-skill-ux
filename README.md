> 将模糊的产品想法，转化为像素级精准的 UI 设计描述。

直接发你的想法——零碎的交互描述、粗糙的页面结构，或者一句「帮我做个弹窗」——wsc-ux 会自动判断场景，整理成结构化规格或直接生成 Figma Make/Stitch 可用的完整提示词。

---

## 它能做什么

### 场景 1：碎片梳理

你发来的描述很零散？直接发过来，根据内容类型自动整理成对应的结构化输出：

**交互状态类** → 输出完整状态机表格

```
状态机：Default / Hover / Active / Loading / Success / Error / Disabled
每个状态包含：触发条件 / 视觉表现 / 动效时长
边界条件：快速连点 debounce / 网络超时 / 空数据态
```

**动画/动效类** → 输出动效时序规格

```
时序：入场 → 中间态 → 退出
参数：属性 / 时长(ms) / 曲线 cubic-bezier
实现要点：需要注意的细节
```

**页面/布局类** → 输出层级拆解文档

```
布局层级：从外到内的模块关系
模块拆解：模块 / 位置 / 内容 / 交互（表格）
信息层级：主要信息 → 次要信息 → 辅助信息
```

**功能想法类** → 输出功能梳理文档

```
核心目标 / 主流程步骤 / 关键决策点 / 边界情况
```

**原则：能推断的直接推断，不确定的标注（推测），不做不必要的追问。**

### 场景 2：Make/Stitch 提示词补全

给一个粗糙的 UI 结构，说「帮我补全」「做个提示词」，自动生成可以直接投喂 Figma AI 插件的完整提示词。

**核心策略：功能逻辑清晰优先，视觉细节留给 AI 自由发挥。**

- ✅ 必须写清楚：模块结构、数据关系、联动逻辑、交互行为、异常状态
- ❌ 不过度指定：具体颜色值、精确像素、圆角数字、动效曲线（除非你明确要求）

**平台判断逻辑：**

| 场景 | 平台 |
| :--- | :--- |
| 完整页面 / 弹窗 / 复杂布局 | Stitch |
| 单个组件 / 卡片 / 按钮 | Make |

**输出结构：**

```
意图解析：一句话说清楚这是什么，核心交互逻辑是什么

[Stitch / Make] 提示词（放入代码块）：
- 视觉规范设定（品牌色 / 栅格 / 圆角 / 字体层级 / 阴影边框）
- 功能与交互说明（模块结构 / 数据关系 / 联动逻辑 / 空态/异常态）
```

### 场景 3：产品顾问（按需加载）

你明确说「帮我分析」「评审一下」「竞品分析」「新功能怎么做」时进入。

弹窗选择模式，选完后加载对应专家文件深度执行：

| 模式 | 适用场景 |
| :--- | :--- |
| **A. 深度分析** | 梳理根因、模块分解、用户反馈归纳 |
| **B. 交互优化** | 现有流程的交互问题诊断与改进 |
| **C. 功能优化** | 已有功能的体验提升与逻辑完善 |
| **D. 新功能设计** | 从零设计一个新功能的完整流程 |
| **E. 方案评审** | 对你已有的设计方案进行专业评审 |
| **F. 竞品分析** | 拆解竞品的设计策略与交互逻辑 |

## 三个核心机制

### 1. 弹窗引导系统（AskUserQuestion）

场景不明确、或需要用户做选择时，统一通过弹窗收集信息，不乱猜也不反复追问纯文字。触发时机：

- 场景 3 进入时，弹出 A~F 模式选择
- 意图模糊时，弹出「你是想梳理交互规格，还是生成提示词？」
- P图/修图意图不清时弹窗确认

---

### 2. 记忆轴系统（Memory）

每次使用后自动将偏好和规律写入本地记忆轴，下次更精准：

| 记忆轴 | 文件 | 记录内容 |
| :--- | :--- | :--- |
| UI 设计偏好 | `memory/ui_design_spec.md` | 视觉风格、布局偏好、组件规律 |
| 交互模式 | `memory/interaction_patterns.md` | 常用交互状态、动效节奏 |
| 项目规律 | `memory/project_patterns.md` | 项目特定规范、品牌色调 |
| 提示词归档 | `memory/ui_prompts.md` | 历史高质量 Make/Stitch 提示词 |

**内置 UI 规范**（自动应用，无需每次说明）：

- 主品牌蓝 `#165DFF`，SaaS 后台视觉风格
- 4px 基准栅格，优先间距：8 / 16 / 24 / 32 / 48px
- 字号层级：页面标题 28px / 模块标题 18px / 正文 14px / 辅助 12px
- 轻量阴影 `0 4px 10px rgba(0,0,0,0.05)`，细边框 `1px solid #E5E6EB`
- 圆角：外层容器 12px，按钮/卡片 8px，标签 4px
- 内容驱动布局，克制留白，信息层级明确

---

### 3. 场景自动分发

不需要你手动告诉它用哪个场景，根据你说的内容自动判断：

```
发来零碎描述（交互 / 动画 / 页面 / 功能）→ 场景 1：碎片梳理
说「帮我补充」「做个提示词」「粗糙结构」→ 场景 2：提示词补全
说「帮我分析」「评审」「竞品」→ 场景 3：产品顾问
```

## 安装与触发

### 安装

将 `wsc-ux` 文件夹放入 Claude Code 的技能目录：

```bash
~/.claude/skills/wsc-ux/
```

### 触发关键词

```
帮我梳理一下 / 补充一下 / 整理一下
这是一个弹窗/页面/交互 / 交互状态 / 动画
UI 界面 / Make / Stitch / 设计稿 / 提示词
帮我分析 / 评审一下 / 竞品分析 / 新功能怎么做
```

- 新增结构化视觉规范模板（8px 栅格 / SaaS 蓝 / 字号层级）内嵌到提示词输出

- 引入场景 3「产品顾问」工作流，支持 A~F 六大专家模式
- 精简碎片梳理逻辑，减少不必要的追问
