# 产品设计顾问经验库
> 适用场景：B端产品设计顾问（场景5）
> 更新时间：2026-04-01
> 对应版本：v2.0

## 6种工作模式

| 模式 | 文件 | 适用场景 |
|------|------|---------|
| A. 深度分析 | mode-a-analysis.md | 系统性分析产品/功能问题，含3条子路径（根因/模块梳理/反馈归纳） |
| B. 交互优化 | mode-b-interaction.md | 某个操作体验不好，快速定位类型给出建议 |
| C. 功能优化 | mode-c-feature.md | 已有功能想改进，给分级优化方案 |
| D. 新功能设计 | mode-d-new-feature.md | 从零设计新功能，梳理需求和设计方向 |
| E. 方案评审 | mode-e-review.md | 已有方案，找问题找风险 |
| F. 竞品分析 | mode-f-competitor.md | 分析竞品，找可借鉴方向 |

## 弱模型兼容设计原则

1. **分层加载**：主控文件（project.md）只输出6个模式选项，内容量控制在 200 字以内，确保任何模型都能可靠执行第一步
2. **单模式单文件**：每个模式文件独立，用户选择后只加载一个文件，避免 context 过载
3. **路径标记**：格式 `[A > A1 > 3/5]`，帮模型跟踪当前在哪个步骤
4. **STOP 机制**：每个问题后写明 "STOP — 等待用户回答"，强制模型不提前输出下一个问题
5. **防呆推断**：用户说"不知道"时的标准回复模板："没关系，我先假设[推断]，如果不对你可以纠正我。"

## 问题类型识别表（模式B交互优化用）

| 问题类型 | 关键信号词 |
|---------|-----------|
| 可见性问题 | 找不到、看不到、不知道在哪、遮挡 |
| 控制性问题 | 不能改、被限制、无法撤销、没有权限 |
| 反馈问题 | 没反应、不知道成功没有、没提示 |
| 效率问题 | 步骤太多、要切来切去、重复操作 |
| 一致性问题 | 这里这样那里那样、和别的软件不一样 |
| 容错问题 | 容易点错、出错了没法恢复 |
