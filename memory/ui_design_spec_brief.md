# UI 设计稿精简规范
> 来源：颜色 / 字体 / 间距圆角阴影 / 视觉决策 / 动效规范提炼
> 适用：Stitch / Figma Make / Cursor / Trae / Copilot 等 AI 生成 UI 场景
> 用途：默认作为 UI 设计稿生成的轻量规范上下文

---

## 一、产品视觉立场

这是一个偏 **SaaS 后台 / 工作台 / 管理系统** 的视觉体系，但不应做成传统拥挤、平均、没有重点的后台。

核心原则：
- **内容驱动布局**，不是组件堆砌布局
- **克制建立张力**，少而准，不花哨
- **留白是设计的一部分**，不要把页面塞满
- **信息层级必须明确**，用户一眼能找到重点
- 视觉气质参考：**现代、克制、清晰、偏高级感**

---

## 二、颜色规范

### 1. 主题模式
- 支持浅色 / 深色双主题
- 组件优先使用**语义化颜色**，不要直接乱用 hex

### 2. 主色规则
- 主品牌蓝：`#165DFF`
- 每页不超过 3 处高强度主色强调
- 主色主要用于：
  - 主要 CTA 按钮
  - 当前选中状态
  - 关键链接 / 重点信息
- 不要大面积把主色当背景底色乱铺

### 3. 语义色
- 成功：绿色
- 警告：橙色
- 错误：红色
- 特殊高亮：紫色

### 4. 页面配色原则
- 页面背景保持干净
- 卡片、面板、页面背景要有层级差，但不要过度对比
- 文本颜色必须分主次层级，不能全一样深
- 图标色默认跟随文本层级，不要到处彩色图标

---

## 三、字体规范

### 1. 字体风格
- 中文 / 英文混排以现代无衬线为主
- 整体偏干净、理性、产品化
- 代码 / 数字 / 数据指标可使用等宽或 tabular numbers 风格

### 2. 字号层级
默认采用跳跃式层级，不要线性堆很多字号。

建议主要层级：
- 页面标题：28px / 500
- 卡片或模块标题：18px / 500
- 正文：14px / 400
- 辅助说明：12px / 400
- 数据大数字：32px 或 40px（按需）

### 3. 规则
- 一个页面常规字号尽量不超过 3 种主层级
- 不要使用 15px、17px 这类模糊中间值
- 标题要有视觉锚点，正文要克制，辅助信息要退后

---

## 四、间距规范

### 1. 核心规则
- 基准单位：**4px**
- 优先使用：8 / 16 / 24 / 32 / 48 / 64
- 必要时使用：4 / 12 / 20 / 28
- 禁止随意出现非 4 倍数间距

### 2. 常用建议
- 行内小间距：8px
- 卡片常规内边距：16px 或 24px
- 模块之间：24px 或 32px
- 页面级留白：32px 起

### 3. 留白原则
- 不要把所有组件挤在一起
- 大标题区、筛选区、数据区要有明显节奏变化
- 重要内容周围可以留更多空白，帮助形成视觉重心

---

## 五、圆角与阴影

### 1. 圆角
- 默认采用中小圆角
- 普通卡片、输入框、按钮风格保持统一
- 不要一个页面同时出现太多种圆角体系

### 2. 阴影
- 阴影用于建立层级，不用于炫技
- 常规卡片阴影要轻
- 弹层、浮层、抽屉可以略强一点
- 不要出现厚重、脏、发灰的阴影

---

## 六、布局与信息层级

### 1. 布局原则
- 页面先有主次，再摆模块
- 不要平均分配所有空间
- 重要内容可以更大、更显眼
- 次要信息应主动弱化

### 2. 信息层级原则
- 页面必须存在明确视觉锚点
- 标题、核心数据、主要操作、辅助说明要形成强弱关系
- 不要所有卡片都一样重
- 不要所有按钮都一样突出

### 3. SaaS / 工作台类页面建议
常见页面模块：
- 页面标题区
- 筛选 / 搜索 / 操作区
- 核心数据区
- 列表 / 表格 / 图表区
- 详情抽屉 / 弹层 / 侧边信息区

---

## 七、基础交互原则

默认生成静态设计稿时，只考虑基础交互暗示，不必展开复杂动效。

基础交互包括：
- hover 状态
- 选中状态
- 禁用状态
- 输入聚焦状态
- 卡片 / 按钮 / 标签的轻反馈

如果用户明确说要：
- 交互稿
- 动效
- 前端实现
- hover / 过渡 / 展开 / 抽屉 / 弹层动画

再额外加载完整动效规范。

---

## 八、设计稿生成时的常见禁忌

不要这样做：
- 页面塞满，没有留白
- 主色滥用，满屏蓝色
- 所有标题和正文都差不多大
- 卡片、列表、按钮没有层级区分
- 图标乱用彩色
- 阴影过重
- 圆角风格混乱
- 一页里塞进太多模块，导致没有重点

---

## 九、默认输出倾向

生成 UI 设计稿时，优先追求：
- 清晰的信息结构
- 克制但高级的视觉风格
- 合理的留白与节奏
- 能直接拿去继续细化或交给 AI 工具生成
