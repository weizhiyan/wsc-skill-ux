# UI 设计稿超轻量规范
> 适用：Stitch / Figma Make / Cursor / Trae 等 AI 生成 UI 场景
> 用途：默认优先加载，尽量少耗 token

## 1. 整体气质
- 面向 SaaS 后台 / 工作台 / 管理系统
- 风格：现代、克制、清晰、有高级感
- 以内容驱动布局，不要堆组件
- 留白是设计的一部分，不要把页面塞满
- 必须有明确的信息层级和视觉重点

## 2. 颜色
- 支持浅色 / 深色双主题
- 主品牌蓝：`#165DFF`
- 主色每页不超过 3 处强强调
- 主色只用于主要按钮、选中态、重点链接
- 页面背景保持干净，卡片和背景要有层级差
- 文本颜色要分主次，图标默认跟随文本层级

## 3. 字体
- 中文界面优先使用 **PingFang SC**
- 英文或数字内容可搭配现代无衬线，但整体风格要与 PingFang 协调
- 默认字号层级：
  - 页面标题：28px / 500
  - 模块标题：18px / 500
  - 正文：14px / 400
  - 辅助说明：12px / 400
- 页面常规字号尽量不超过 3 种主层级
- 不要用 15px、17px 这类中间值

## 4. 间距
- 基准单位：4px
- 优先用：8 / 16 / 24 / 32 / 48 / 64
- 卡片内边距常用：16px 或 24px
- 模块间距常用：24px 或 32px
- 页面级留白：32px 起
- 不要出现非 4 倍数间距

## 5. 圆角与阴影
- 使用统一的中小圆角
- 阴影只用于建立层级，不要厚重
- 卡片阴影轻，浮层可略强
- 不要混用太多圆角和阴影风格

## 6. 布局与层级
- 页面先定主次，再摆模块
- 优先使用自动布局（Auto Layout）组织页面与组件
- 不要为了包裹而额外套多余 Frame
- 结构要干净，层级尽量少但清晰
- 页面内容要尽量填充合理区域，避免空洞和松散
- 不要平均分配所有空间
- 重要内容可以更大、更显眼
- 不要所有卡片、按钮、模块一样重
- 常见页面结构：标题区 / 筛选操作区 / 核心数据区 / 列表图表区 / 详情侧栏或弹层

## 7. 基础交互
- 默认只考虑基础交互暗示：hover、选中、禁用、聚焦
- 若用户明确要交互稿 / 动效 / 前端实现，再加载完整动效规范

## 8. 禁忌
- 不要满屏主色
- 不要页面过满、没有留白
- 不要所有文字一样大
- 不要图标到处彩色
- 不要阴影过重、圆角混乱
- 不要一页塞太多模块导致没有重点
